FENDER

Forensic Extraction of Navigational Data & Event Records

A powerful forensic tool for extracting GPS location data from vehicle telematics binary files. Supports multiple manufacturers with an easy-to-use interface for investigators and researchers.

📥 Download FENDER 🔗 View on GitHub

Features

🚗

Multi-Manufacturer Support

Extract GPS data from Honda, Toyota, OnStar, Mercedes-Benz, and Stellantis vehicle systems with specialized decoders for each platform.

📊

Multiple Export Formats

Export your findings to XLSX, JSON, and KML formats for analysis in your preferred tools or mapping applications.

🖥️

GUI & CLI Modes

Choose between an intuitive graphical interface for ease of use or command-line interface for automation and batch processing.

🔒

Secure File Handling

Built with forensic best practices including secure file operations, integrity verification, and comprehensive logging.

🔌

Plugin Architecture

Modular decoder system allows for easy addition of new vehicle manufacturers and telematics systems.

📍

Comprehensive Data

Extract latitude, longitude, timestamps, and additional metadata like speed, heading, and trip information where available.

Installation

🪟 Windows Users

Download the pre-built executable for easy installation:


  1. Download FENDER.exe from the releases page
  2. Double-click to run the application
  3. If Windows Defender shows a warning, click "More info" and "Run anyway"

🐍 Python Users

Install from source for the latest features:

# Clone the repository
git clone https://github.com/BitEU/FENDER.git
cd FENDER

# Install dependencies
pip install -r requirements.txt

# Run FENDER
python main.py

📋 System Requirements

  • Windows 10/11 (for .exe release)
  • Python 3.8+ (for source code)
  • 4GB RAM minimum
  • 500MB free disk space
